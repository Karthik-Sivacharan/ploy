# Ploy — Product Brainstorm

## The Problem

A typical marketing team juggles 10-15 tools that don't talk to each other:

- **Brand**: Style guides in Frontify/Notion, assets in Google Drive/Dropbox, brand guidelines in a PDF nobody reads
- **Website**: Webflow/WordPress for pages, separate CMS for blog, another tool for landing pages
- **Growth channels**: Mailchimp for email, Buffer for social, Google Ads console, LinkedIn campaign manager, HubSpot for sequences

The result: brand inconsistency, copy-paste workflows, context switching. Marketing teams spend more time operating tools than doing marketing. Every new campaign means touching 5+ tools, manually keeping them in sync.

## The Insight (Why Now)

AI changes the game — but only if the platform is built AI-native. You can't just bolt AI onto HubSpot. If AI can generate copy, design pages, and orchestrate campaigns, then the tool boundaries become the bottleneck, not human productivity. Collapse all the tools into one surface and let AI work across the whole surface.

Bryant's unique angle: he built the best visual website builder (Webflow) AND ran growth/marketing there. He's lived both sides.

---

## Three Interconnected Layers

### 1. Brand Layer (the foundation)
- Your brand lives here: colors, typography, voice/tone guidelines, logos, imagery style, key messaging
- Not a static style guide — a living system that AI references for everything it generates
- AI enforces brand consistency across every output automatically

### 2. Website Layer (the storefront)
- Visual website builder (Bryant knows how to build this better than anyone)
- Integrated with brand and growth — not a separate product
- Landing pages, blog, main site, microsites — all in one place
- AI can generate on-brand pages from a brief

### 3. Growth Layer (the engine)
- Multi-channel campaign orchestration: email, social, ads, SEO, outbound
- One workflow that outputs to all channels
- AI generates channel-appropriate content from a single campaign brief, all on-brand
- Built-in analytics showing performance across channels

---

## Core Workflow

> "Launch a product announcement"
>
> Write a brief (or tell AI). Ploy then:
> - Drafts a landing page using your brand system
> - Generates email sequences for different segments
> - Creates social posts sized for each platform
> - Suggests ad copy and targeting
> - Everything matches brand voice and visual identity
> - Review, tweak, approve — all goes live from one place
> - Performance data flows back into one dashboard

Today: team coordinating across 6 tools over 2 weeks. With Ploy: a single afternoon.

- Bryant's credibility in visual web building + growth

## Direction: Visual Workflow Builder + AI Agent

Could Ploy be a **visual workflow builder** (like n8n/sim.ai/weavy.ai) where:
- AI agent builds a marketing workflow based on a campaign brief or project
- The workflow is visualized as a node-based canvas
- Marketing person can see each step, tweak content/targeting/timing
- Then launch the whole thing from one place

This would combine:
- The **agentic AI** trend (AI does the heavy lifting)
- **Visual transparency** (marketer sees and controls what's happening)
- **Human-in-the-loop** (tweak before launch, not blind automation)
